# Simple Archiver

> Небольшой терминальный архиватор на Go с собственным форматом `.sarch`, простым RLE-сжатием и интерфейсом на Bubble Tea.

`Simple Archiver` — это учебный проект, в котором реализованы:
- работа с файлами через Go;
- собственный формат архива;
- базовое сжатие и распаковка;
- текстовый интерфейс в терминале.

Проект позволяет выбрать действие в меню, ввести путь к файлу и выполнить сжатие или распаковку прямо из консоли.

---

## ✨ Возможности

- Сжатие файла в архив формата `.sarch`
- Распаковка архива обратно в директорию исходного файла
- Интерактивное меню в терминале
- Экран ввода пути с курсором
- Вывод ошибок прямо в интерфейсе
- Навигация с клавиатуры без дополнительных аргументов командной строки

## 🎮 Использование

После запуска откроется главное меню:

```text
=== Простой архиватор ===

> Сжать файл
  Распаковать файл
  Выход
```

### Сжатие файла

1. Выберите пункт `Сжать файл`
2. Нажмите `Enter`
3. Введите путь к файлу, например `test.txt`
4. Нажмите `Enter`
5. Программа создаст архив `test.txt.sarch`

Пример экрана:

```text
=== Простой архиватор ===

Введите путь к файлу для сжатия:
test.txt_

Enter для подтверждения, Esc для возврата в меню
```

### Распаковка файла

1. Выберите пункт `Распаковать файл`
2. Нажмите `Enter`
3. Введите путь к архиву, например `test.txt.sarch`
4. Нажмите `Enter`
5. Программа распакует файл в директорию, где находится архив

Пример экрана:

```text
=== Простой архиватор ===

Введите путь к файлу для распаковки:
test.txt.sarch_

Enter для подтверждения, Esc для возврата в меню
```

## ⌨️ Управление

- `↑` / `k` — перемещение вверх по меню
- `↓` / `j` — перемещение вниз по меню
- `Enter` — выбрать пункт меню или подтвердить путь
- `Backspace` — удалить последний символ
- `Esc` — вернуться в главное меню
- `q` / `Ctrl+C` — выйти из программы

---

## ❗ Обработка ошибок

Если операция завершилась с ошибкой, приложение остаётся на экране ввода и показывает сообщение пользователю. Ошибка хранится в состоянии модели и отображается через экран ввода пути.

Пример:

```text
Ошибка: open input file "nonexistent.txt": open nonexistent.txt: no such file or directory
```

---

## 📦 Формат архива

Архив создаётся в собственном формате:

1. 1 байт — длина имени исходного файла
2. N байт — имя исходного файла
3. Далее идут блоки данных
4. Каждый блок начинается с 2 байт размера сжатого блока
5. После этого записываются сами данные блока

При распаковке программа сначала читает имя файла из заголовка архива, затем создаёт выходной файл в целевой директории и последовательно восстанавливает данные из блоков.
